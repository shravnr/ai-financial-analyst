Eval Framework — Flow

1 · Setup
Run eval
python -m src.evaluation.run_eval
Auto-ingest tickers if not indexed
Deep eval: AMZN  |  Smoke: random from {AAPL, MSFT, GOOGL}  |  EDGAR download → chunk → embed → ChromaDB
2 · Deep Eval (AMZN) — LLM-as-judge
Run templatized questions through the live system
Questions use {company} placeholder — works for any ticker without rewriting
Signal 1: Consistency
4 question pairs — same question, different phrasing → ask gpt-5.4-mini: do both answers agree? Score 1–5 per pair
Signal 2: Grounding
10 capability questions → ask gpt-5.4-mini: is every claim supported by the retrieved context? Score 1–5 per question
Signal 3: Boundary
Questions the system must refuse (future price prediction, unknown ticker) → check: no fabricated $ amounts + classified as refusal
3 · Rule-based signals
Signal 4: Verification
Citation check: does every answer have [N] citations?  |  Number check: every $ amount traceable to source within 1%?
Signal 5: Smoke test
Basic questions on second ticker (random from pool) → pass if citations present  |  flags thin context (≤1 source)
4 · Scoring
Normalize all signals to 0–10
LLM-judge avg: (Consistency + Grounding + Boundary) / 3  |  Rule-based avg: (Verification + Smoke) / 2
Master score = avg(LLM-judge, Rule-based)
AMZN: 9.7 / 10  |  AAPL: 9.5 / 10
Decision
Master score ≥ 8.5?
PASS ✓
Save results to eval_results/ → acts as regression gate for future changes
FAIL ✗
Print failures & flags → diagnose which signal dropped → fix retrieval / prompts / guardrails
